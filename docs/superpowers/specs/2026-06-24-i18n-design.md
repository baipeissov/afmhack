# i18n для фронтенда (RU / KK / EN)

## Цель

Добавить локализацию интерфейса (UI-оболочка, не контент мок-данных) на русский,
казахский и английский языки с точным, профессиональным переводом — особенно для
казахского, где нужна корректная финансовая/AML-терминология, а не машинный перевод.

## Языки и роутинг

- Локали: `ru` (default), `kk`, `en`
- URL с префиксом локали: `/ru/command`, `/kk/command`, `/en/command`
- Корень `/` редиректит на `/ru` (дефолтный язык, не auto-detect по Accept-Language)
- Выбор языка сохраняется в cookie `NEXT_LOCALE` между визитами

## Библиотека

`next-intl` — нативная поддержка Next.js App Router (Server + Client Components),
ICU message format, типобезопасные namespace-словари, встроенный middleware для
locale-routing. Альтернативы (react-i18next, самописный context) хуже стыкуются с
App Router/Server Components и требуют больше ручной интеграции — не рассматриваем.

## Структура файлов

```
src/
  middleware.ts                 # next-intl middleware, matcher на все страницы кроме /api, /_next, статики
  i18n/
    routing.ts                  # defineRouting({ locales: ["ru","kk","en"], defaultLocale: "ru" })
    request.ts                  # getRequestConfig — подгрузка messages/<locale>.json
  app/
    [locale]/
      layout.tsx                # NextIntlClientProvider, оборачивает текущий root layout
      (app)/...                 # все существующие страницы переезжают сюда без изменения структуры
      page.tsx                  # redirect("/command") как сейчас, но относительно [locale]
messages/
  ru.json
  kk.json
  en.json
```

Существующий `src/app/page.tsx` (редирект на `/command`) и весь `src/app/(app)/**`
переносятся под `src/app/[locale]/`. Структура папок `(app)/...` внутри не меняется.

## Namespace словарей

Один JSON-файл на язык, разбитый на ключи-неймспейсы:

```
common      — общие слова (Save, Cancel, Loading, Search…)
nav         — пункты NavRail (Command, Detections, Investigations, Entities, Graph,
              Analytics, Agents, Vault, Reports, Admin, "New case", "system nominal")
topbar      — поиск, "agents active", языковой переключатель
command     — страница /command
detections  — страница /detections
investigations — список + вложенные вкладки (agents, contradictions, graph, report, timeline)
entities    — страница /entities
analytics   — страница /analytics
agents      — страница /agents
evidence    — страница /evidence (Vault)
reports     — страница /reports
settings    — страница /settings
components  — общие переиспользуемые виджеты (severity-badge, empty-state,
              confidence-meter, kpi-stat, case-header, case-tabs, verdict-meter,
              how-it-works, status-strip)
```

## Переключатель языка

Новый клиентский компонент `src/components/shell/language-switcher.tsx`:
компактный dropdown (RU / KK / EN) в `TopBar`, рядом с часами. При выборе языка —
переход на тот же путь с новым префиксом локали через `next-intl`'s
locale-aware `Link`/`useRouter` (сохраняет динамические сегменты, например
`/investigations/CASE-2041/graph`).

## Качество перевода

- RU — исходный язык контента (текущий английский UI переводится на естественный
  деловой русский, без калек)
- KK — профессиональный перевод с учётом сложной морфологии казахского
  (агглютинация, гармония гласных, падежные формы), с использованием устоявшейся
  финансовой/AML-терминологии, принятой в банковском и госсекторе РК (АФМ и т.п.),
  а не пословный машинный перевод
- EN — текущие строки остаются почти как есть (это исходный язык интерфейса),
  при необходимости приводятся к единому стилю

## Вне скоупа

- Перевод содержимого `src/mocks/*` (кейсы, транскрипты, evidence-тексты)
- RTL-поддержка (не нужна для ru/kk/en)
- Серверная локализация данных из будущего backend API (api/, pipeline/) — это
  отдельная задача, не часть текущего фронтенд-скоупа
